306  // Special values.
307  // The unguaranteed order of initialization of static variables across
308  // translation units can be a source of annoying bugs, so now the static
309  // special values (like Vector3::ZERO, Color3::WHITE, ...) are wrapped
310  // inside static functions that return references to them.
311  // These functions are intentionally not inlined, because:
312  // "You might be tempted to write [...] them as inline functions
313  // inside their respective header files, but this is something you
314  // must definitely not do. An inline function can be duplicated
315  // in every file in which it appears and this duplication
316  // includes the static object definition. Because inline functions
317  // automatically default to internal linkage, this would result in
318  // having multiple static objects across the various translation
319  // units, which would certainly cause problems. So you must
320  // ensure that there is only one definition of each wrapping
321  // function, and this means not making the wrapping functions inline",
